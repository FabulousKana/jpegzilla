About Jpegzilla

MozJPEG lacks a good user-friendly front-end, that's why Jpegzilla has been made. Jpegzilla is open-source software, it has been confirmed it can run on Windows XP+ and every modern Linux distribution. It requires Python 3.4+ installed (or library bundled with the binary). It uses the Tkinter library for GUI as it is lightweight, fast and simple.

Usage

Functions

  • Image Quality - Compression quality (5-95 is the most useful range)
  • Smoothing - Smooth dithered input (1..100 is strength)
  • Progressive - Create progressive JPEG file (on by default)
  • Greyscale - Create monochrome JPEG file
  • Use arithmetic coding - en.wikipedia.org/wiki/Arithetic_coding
  • Optimize - Optimize Huffman table (smaller file, but slow compression, on by default)
  • Baseline - Create baseline JPEG file (incompatible with Progressive)
  • Disable trellis optimization - en.wikipedia.org/wiki/Trellis_quantization
  • Rotate - Rotate image (degrees clockwise)
  • Transpose - (incompatible with Transverse)
  • Transverse - (incompatible with Transpose)
  • Drop non-transformable edge blocks
  • Crop to a rectangular subarea - syntax WIDTHxHEIGHT+X_AXIS+Y_AXIS, example: 500x300+200+100

More information can be found in jpegtran manual.
